# Decisions

## 2026-07-05 문서 구조 분리

- 결정: 개발 문서를 `docs/` 아래 역할별 Markdown으로 분리한다.
- 이유: Codex와 사람이 필요한 문서만 참고하기 쉽고, 기획/디자인/개발/QA 기준이 섞이지 않는다.

## 2026-07-05 AGENTS.md 사용

- 결정: 사용자가 말한 `agentd.md`는 별칭 문서로 두고, 실제 규칙은 표준적인 `AGENTS.md`에 둔다.
- 이유: Codex 계열 도구가 `AGENTS.md`를 프로젝트 작업 규칙으로 인식하기 쉽다.

## 2026-07-05 원페이지 우선

- 결정: 초기 버전은 반응형 원페이지로 만든다.
- 이유: 에어컨 청소 서비스는 고객이 빠르게 신뢰 요소와 예약 CTA까지 이동하는 흐름이 중요하다.
